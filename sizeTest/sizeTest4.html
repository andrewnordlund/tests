Thoughts on WCAG 1.4.4 - Resize Text (Level AA)

Assumptions and Assertions

  1. I'm assuming that in order for a Success Criterion to be meaningful, there has to be a way for it to not be met - it has to be able to fail.
  2. Given that WCAG 4.1.1 - Parsing is likely to be deprecated in WCAG 2.2, then I'm assuming that if a success criterion is no longer an issue (due to advancements in technology, etc.) then it will be deprecated.
  3. Furthermore, when interpreting other's writings, it is important that if an interpreter is going to insist on one interpretation they need to be able to provide alternate wording for any alternate interpretations. (i.e.: "If they meant to say x, they would have said it this way....")
