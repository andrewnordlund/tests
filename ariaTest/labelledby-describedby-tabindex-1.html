Summary

  • Using aria-describedby or aria-labelledby should only be used in the <fieldset> element, and not the <legend>.
  • Having aria-describedby in the <fieldset> element enjoys the most support; still lacks with IE, VoiceOver + non-Safari, and Narrator + non-Edge.
  • Even though Edge and Chrome use the same engine (Chromium) Narrator only reads <legend> text with Edge. It's like they went out of their way to not support other browsers.
  • Putting aria-labelledby in the <fieldset> element has the most consistent support: it reads the paragraph instead of the <legend> text. The only stacks this doesn't apply to are: NVDA + Internet Explorer and Narrator + non-Edge.
  • Putting aria-describedby in the <legend> element is never supported. The paragraph is just ignored.
  • Support for putting aria-labelledby in the <legend> is all over the map. Don't do.
  • VoiceOver on iOS has poor <legend> support in the first place. So, even though having aria-describedby in the <fieldset> element only reads the paragraph, and not the <legend> text, that's consistent with how it handles <legend>s without aria-describedby. Maybe this will be fixed in iOS 15?
  • Adding tabindex="-1" to the Extra Text makes no difference whatsoever. (See test results.) (I think it used to make a difference. See: Testing aria-describedby and aria-labelledby in browsers and screen readers. Does it require tabindex on span references in IE11?- David MacDonald Web Accessibility Blog, testing, teaching, and discussion.)

Conclusion

Dos and don'ts:

  1. Don't put aria-describedby or aria-labelledby in a <legend> element.
  2. Do use aria-describedby in <input>s to supplement the <label>.
  3. Do use Safari if using OS X with VoiceOver. Firefox support is getting better, but Safari is still your best bet.
  4. Do use Edge if you must use Narrator.
  5. Only use aria-describedby in the <fieldset> element if you don't care about supporting Internet Explorer or non-supported stacks. (VoiceOver only officially supports Safari on OS X, and apparently Narrator only supports Edge on Windows 10.)
