Screen Reader Results

VoiceOver Mac OS X
Safari, Chrome, Firefox
In browse mode (control+option+arrow keys) the psuedo-content is always read if it is visible ("star stuff...".) When tabbing, it's read when it's visible, except on <fieldset>. But it is read as part of the <legend>.
VoiceOver oOS
Safari
In browse mode (swipe left/right) the psuedo-content is always read if it is visible ("star stuff...".) When using the rotor (links/form controls), it's read when it's visible, except on <fieldset> even when it's part of the <legend>. But this is a VoiceOver iOS issue; the <legend> is never read when swiping up/down with rotor on Form Controls.
NVDA 2021.3.1
Firefox
In browse mode, the psuedo-content is always read if it's visible. When you get to the <fieldset>, first it reads "group start" then when you press down arrow-key it says "star stuff". When it's applied to the <legend> you hear "grouping start star stuff pick some" without having to hit down arrow-key. In Forms Mode (tabbing through the page) it's read in all places except with it's applied to <fieldset>, but it is read when aplied to <legend.> as part of the <legend.> text.
Chrome
In browse mode, the psuedo-content is always read if it's visible. When you get to the <fieldset>, first it reads "group start" then when you press down arrow-key it says "star stuff". When it's applied to the <legend> you hear "grouping start star stuff pick some" without having to hit down arrow-key. In Forms Mode (tabbing through the page) it's read in all places except with it's applied to either <fieldset> or <legend.>.
