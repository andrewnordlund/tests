Summary

  1. When using OS X, Firefox has the best support. Chrome is all over the map. And Safari is decent, but still lacking.
  2. When using OS X, if you have any row headings, it's best to stick with id/headers. scope attributes should be best left to column headers.
  3. There's pretty decent support for iOS 14 when there's a single row of <th>s. Then things get mildly wonky with a column of row headers.
  4. Narrator sucks.
  5. NVDA and JAWS are pretty consistent across the board. They work with implicit headings, even when there's a row and column header.

Recommendations

There was lots that wasn't tested here, like more complex tables with multiple rows and columns of headers. My old rule-of-thumb that if each <td> only has 1 <th>, then use scope, else use id/headers seems to be now outdated. In fact, using scope with a row and column of headers works better with Narrator and IE. Other than that, id/headers offers no advantages. This suprised me. Perhaps with more complex tables, id/headers will be more useful.
